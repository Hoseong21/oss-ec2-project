| 소속 | 광운대학교 수학과 (정보융합학부 복수전공) |

## 프로젝트 소개

Career Compass는 정보융합학부 학생이 본인의 진로 성향을 파악하고, 학과 이수체계도와 연결된 맞춤 교과목 가이드를 받을 수 있는 Streamlit 기반 웹 애플리케이션입니다.

21문항의 2지선다 테스트로 학생의 성향을 3개 축(관심 영역 / 문제 접근 방식 / 선호 환경)으로 분석하고, 그 조합으로 8가지 진로 유형 중 하나를 도출합니다. 각 유형마다 상세 설명, 빠지기 쉬운 함정, 추천 직무, 그리고 학과 2026 이수체계도 위에 표시된 추천 과목을 제공합니다.

기존 진로 적성 테스트가 결과를 모호하게만 알려주는 한계를 보완하기 위해, 결과를 **학과 교과목 선택과 직접 연결**하는 데 초점을 맞췄습니다.


## 주요 기능

| 기능 | 설명 |
|---|---|
| 첫 화면 학번/이름 | 앱 진입 시 제출자(개발자)의 학번과 이름이 표시됩니다 |
| 학번 기반 로그인 | 학번 10자리 + 이름으로 사용자를 식별하며, 신규 사용자는 모달로 등록 여부를 확인합니다 |
| 21문항 테스트 | 3개 Part(관심 영역 / 문제 접근 방식 / 선호 환경)로 나뉜 2지선다 문항을 진행합니다 |
| 8유형 진단 | 3축 점수를 백분율로 계산하여 8가지 유형 중 하나로 분류하고, 유형별 상세 설명·함정·추천 직무를 제공합니다 |
| 맞춤 이수체계도 | 본인 유형에 맞는 추천 과목을 학과 2026 이수체계도 위에 핵심/보조/그 외 3단계로 시각화합니다 |
| 테스트 기록 관리 | 사용자별 테스트 기록을 저장하고, 이전 결과를 다시 확인하거나 삭제할 수 있습니다 |
| 캐싱 효과 시연 | 결과 페이지 하단에서 캐싱 적용 함수의 실행 시간 차이를 직접 측정해볼 수 있습니다 |


## 실행 방법

### 요구 사항
- Python 3.9+ (개발자는 3.9.6 사용)

### 설치 및 실행

```bash
# 1. 저장소 클론
git clone https://github.com/Hoseong21/oss-streamlit-project.git
cd oss-streamlit-project

# 2. 가상환경 생성 및 활성화 (권장)
python -m venv .venv
source .venv/bin/activate    # macOS/Linux
# .venv\Scripts\activate     # Windows

# 3. 의존성 설치
pip install -r requirements.txt

# 4. 실행
streamlit run app.py
```

실행 후 브라우저에서 자동으로 `http://localhost:8501` 페이지가 열립니다.


## 8유형 시스템

본 앱은 학생의 성향을 3개 축으로 분석하여 2³ = 8가지 유형 중 하나로 분류합니다.

| 축 | 양극 | 의미 |
|---|---|---|
| **D / V** | Data ↔ Visual | 데이터사이언스 vs 비주얼테크놀로지 — 어떤 분야에 끌리는지 |
| **A / P** | Analytical ↔ Practical | 원리·이론 탐구 vs 실험·구현 중심 — 문제를 어떻게 접근하는지 |
| **R / S** | Research ↔ Service | 연구 환경 vs 서비스 환경 — 어떤 환경에서 일하고 싶은지 |

3개 축 점수의 우세한 쪽 조합으로 8가지 유형 중 하나로 결과가 도출됩니다. 각 유형마다 상세 설명, 빠지기 쉬운 함정, 추천 직무, 그리고 학과 이수체계도 위에 표시된 추천 과목이 함께 제공됩니다.


## 점수 계산 방식

1\. **문항 구성**: 21문항이 3축에 7문항씩 균등 배분되어 있습니다 (D_V축 7문항, A_P축 7문항, R_S축 7문항)  
2\. **선택지 매핑**: 각 문항의 2개 선택지는 해당 축의 두 극(예: D 또는 V)에 매핑됩니다  
3\. **점수 합산**: 사용자가 선택한 답변을 축별로 합산합니다  
4\. **백분율 변환**: 각 축에서 두 극의 점수를 비율로 변환합니다 (예: D 5점 / V 2점 → D 71% / V 29%)  
5\. **유형 결정**: 3축에서 우세한 쪽을 조합하여 유형 코드를 도출합니다 (예: D + A + R → DAR)  


## 캐싱 적용 위치
Streamlit은 사용자 입력마다 전체 스크립트를 재실행하므로, 동일한 데이터를 반복 로드하거나 동일한 차트를 반복 생성하면 누적 비용이 커집니다. 본 앱은 이를 방지하기 위해 4개 함수에 `@st.cache_data`를 적용했습니다.

| 함수 | 위치 | 역할 |
|---|---|---|
| `load_questions()` | `modules/data_loader.py` | 21문항 + 축 매핑 JSON 로드 |
| `load_results()` | `modules/data_loader.py` | 8유형 콘텐츠 JSON 로드 |
| `load_courses()` | `modules/data_loader.py` | 43과목 + 추천 매핑 JSON 로드 |
| `build_roadmap_figure()` | `modules/roadmap.py` | 이수체계도 Plotly 차트 객체 생성 |

> 결과 페이지 하단의 **'직접 확인해보기'** 섹션에서 각 함수의 캐싱 ON/OFF 실행 시간 차이를 직접 측정해볼 수 있습니다.


## 파일 구조

```
oss-streamlit-project/
├── app.py                  # Streamlit 메인 실행 파일
├── requirements.txt        # 의존성 목록
├── .gitignore              # Python 프로젝트용 gitignore
├── README.md               # 프로젝트 설명 (현재 파일)
├── modules/
│   ├── __init__.py         # 패키지 초기화
│   ├── auth.py             # 학번 기반 사용자 인증 및 기록 관리
│   ├── data_loader.py      # JSON 데이터 로더 (캐싱 적용)
│   ├── quiz.py             # 점수 계산 및 유형 도출 로직
│   ├── result.py           # 결과 화면 렌더링
│   └── roadmap.py          # 이수체계도 시각화 (캐싱 적용)
├── data/
│   ├── questions.json      # 21문항 + 축 매핑
│   ├── results.json        # 8유형 콘텐츠 (설명, 함정, 직무)
│   └── courses.json        # 43과목 + 8유형별 추천 매핑
├── docs/
│   └── questions_metadata.json  # 문항 설계 근거 문서 (자기검증 기록)
└── users/                  # 사용자별 테스트 기록 (앱 실행 시 자동 생성, gitignore 처리)
    └── {학번}.json
```
